# Phase 4 Contracts

## Datamap builder contract

Phase 4 adds the database-aware sigilo entry points in:

- [`atlas/sigilo/datamap.py`](../../atlas/sigilo/datamap.py)
- [`atlas/sigilo/style.py`](../../atlas/sigilo/style.py)
- [`atlas/sigilo/hover.py`](../../atlas/sigilo/hover.py)

The public rendering surface now includes:

- `DatamapSigiloBuilder.from_introspection_result(result)`
- `DatamapSigiloBuilder.set_schema_filter([...])`
- `DatamapSigiloBuilder.set_size_scale("log" | "linear" | "sqrt")`
- `DatamapSigiloBuilder.set_style("network" | "seal" | "compact")`
- `DatamapSigiloBuilder.set_layout("circular" | "force")`
- `DatamapSigiloBuilder.set_force_params(iterations, temperature, cooling)`
- `DatamapSigiloBuilder.build() -> bytes`
- `SigiloStyle`
- `get_style_params(style)`

`DatamapSigiloBuilder` converts `IntrospectionResult` into enriched
`SigiloNode` and `SigiloEdge` collections before delegating rendering to the
generic `SigiloBuilder`.

## SVG structure contract

Rendered database datamaps now emit:

- node wrappers as `<g class="system-node-wrap" ...>`
- edge wrappers as `<g class="system-edge-wrap" ...>`
- node-level `data-table`, `data-schema`, `data-row-estimate`,
  `data-size-bytes`, `data-column-count`, `data-fk-count`,
  `data-index-count`, `data-table-type`, and `data-comment`
- edge-level `data-fk-from`, `data-fk-to`, `data-fk-columns`,
  `data-fk-type`, and `data-on-delete` when present
- primary-key dots as `.col-pk`
- declared FKs as `.call`
- inferred FKs as `.branch`
- views as `.node-aux`
- materialized views as `.node-loop`

Node ids generated by the datamap builder use the normalized
`<schema>.<table>` form produced by replacing internal dots and dashes with
underscores before joining the two parts.

## Style and sizing contract

`SigiloConfig` and the native ABI now carry the Phase 4 style fields:

- `canvas_w`, `canvas_h`
- `emit_macro_rings`
- `ring_opacity`
- `ring_stroke_dash`
- `schema_orbit_r`
- `schema_ring_r_base`
- `font_label`
- `font_hash`

Style presets currently resolve to:

- `network`: `1200x1200`, visible macro-rings
- `seal`: `512x512`, dashed macro-rings
- `compact`: `800x800`, macro-rings disabled

`SigiloNode` now carries `fk_count`, `index_count`, `r`, `cx`, and `cy`.
`r` is treated as a precomputed radius hint by both the Python fallback and the
native renderer.

## Hover metadata contract

The final SVG is post-processed by
[`atlas/sigilo/builder.py`](../../atlas/sigilo/builder.py)
to inject the script generated by
[`atlas/sigilo/hover.py`](../../atlas/sigilo/hover.py).

The injected block:

- is appended immediately before the closing `</svg>`
- is emitted as `<script type="text/javascript"><![CDATA[ ... ]]></script>`
- creates a reusable tooltip group with id `atlas-tooltip`
- binds hover behavior to `.system-node-wrap` and `.system-edge-wrap`
- formats row estimates and byte sizes inside the browser

Operational limit:

- interactive hover requires an SVG viewer that executes embedded JavaScript;
  static renderers still retain `<title>` metadata and `data-*` attributes

## Native force-layout contract

The native ABI expands in:

- [`atlas/_c/atlas_render.h`](../../atlas/_c/atlas_render.h)
- [`atlas/_c/atlas_render.c`](../../atlas/_c/atlas_render.c)
- [`atlas/_c/atlas_sigilo.h`](../../atlas/_c/atlas_sigilo.h)
- [`atlas/_sigilo_build.py`](../../atlas/_sigilo_build.py)
- [`atlas/_sigilo.py`](../../atlas/_sigilo.py)

Additional ABI fields:

- `atlas_node_t.fk_count`
- `atlas_node_t.index_count`
- `atlas_node_t.schema_group_idx`
- `atlas_edge_t.from_idx`
- `atlas_edge_t.to_idx`

Additional exported function:

- `atlas_render_compute_layout_force(ctx, iterations, temperature_start, cooling_rate)`

Python access:

- `RenderContext.compute_layout_force(iterations=300, temperature=1.0, cooling=0.98)`

Behavior:

- `DatamapSigiloBuilder.set_layout("force")` requests the native force layout
- graphs with five or fewer nodes fall back to circular layout
- the native implementation starts from the circular layout seed, applies
  repulsion, FK attraction, schema cohesion, and center attraction, then
  refreshes schema-ring geometry from the converged positions

Operational limits:

- the Python fallback keeps a simplified deterministic force relaxation instead
  of the full native simulation
- the force layout is optimized for readable exploration maps, not for exact
  edge-minimization guarantees on extremely dense graphs
